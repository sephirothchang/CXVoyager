# STEP 14 - PERF_RELIABILITY

目的
- 执行性能与可靠性验证场景（例如并发 I/O 基准、故障注入、恢复测试），并生成评估报告。

输入
- `ctx.plan` 与 `ctx.extra` 中的部署/主机信息
- 测试场景配置（fio 场景、故障注入策略等）

输出
- `ctx.extra['perf_reliability']`：测试结果摘要、报告路径与指标时间序列

关键模块
- `cxvoyager.core.deployment.handlers.perf_reliability`：包含场景构建、并发执行与结果采集逻辑。
- 可能依赖外部基准工具（FIO 等）以及资源调度模块

主要流程
1. 根据配置与规划表选择测试场景与受试主机。
2. 并发下发任务（fio、网络吞吐、延迟测试）并收集指标。
3. 可选执行故障注入（断网/拔盘/主机重启），并监测系统恢复能力。
4. 汇总结果、生成报告并将指标写入阶段输出。

容错与注意事项
- 性能测试对系统影响较大，请在非生产或授权环境运行。
- 故障注入必须有明确的回滚与安全策略，避免产生不可逆损失。

日志与可观测
- 记录每个测试子任务的起止时间与原始输出，便于复现与分析。
